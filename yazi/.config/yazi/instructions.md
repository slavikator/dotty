### Інструкція: Як налаштувати Yazi для відкриття файлів у Neovim

Я проаналізував ваш конфігураційний файл. Все налаштовано у файлі `yazi.toml` в секції `[opener]`.

Ось як це працює:
-   `edit = [ ... ]` — цей рядок змушує `yazi` відкривати файл в `nvim` для редагування, коли ви натискаєте клавішу `e`.
-   `open = [ ... ]` — цей рядок змушує `yazi` відкривати будь-який текстовий файл (`mime = "text/*"`) в `nvim`, коли ви натискаєте `Enter` або `l`.

#### Кроки для налаштування:

1.  **Створіть директорію для конфігурації**, якщо її ще немає. Відкрийте термінал і виконайте команду:
    ```bash
    mkdir -p ~/.config/yazi
    ```

2.  **Створіть та відкрийте конфігураційний файл** `yazi.toml` за допомогою `nvim` (або будь-якого іншого текстового редактора):
    ```bash
    nvim ~/.config/yazi/yazi.toml
    ```

3.  **Додайте наступний текст** у файл `yazi.toml`:

    ```toml
    # Секція, що визначає, які програми використовувати для відкриття файлів.
    [opener]

    # Правило для дії "редагувати" (клавіша 'e').
    # Буде відкривати будь-який файл у Neovim.
    edit = [
        { run = 'nvim "$@"', block = true },
    ]

    # Правило для стандартної дії "відкрити" (клавіші Enter або 'l').
    # Буде відкривати будь-який текстовий файл у Neovim.
    open = [
        { run = 'nvim "$@"', block = true, mime = "text/*" },
    ]
    ```

4.  **Збережіть файл та вийдіть** з редактора.

Тепер `yazi` буде використовувати `nvim` для відкриття та редагування текстових файлів.

**Пояснення опцій:**
-   `run = 'nvim "$@"'`: Команда, яку потрібно виконати. `"$@"` — це спеціальний символ, який `yazi` замінює на шлях до вибраного файлу.
-   `block = true`: `yazi` чекатиме, доки ви не закриєте `nvim`, перш ніж ви зможете знову користуватися файловим менеджером.
-   `mime = "text/*"`: Застосовує правило лише до файлів, які система розпізнає як текстові (наприклад, `.txt`, `.md`, `.py`, `.js` тощо).
